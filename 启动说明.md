# Daily Health - 个人健康饮食记录系统

## 📋 项目概述

这是一个完整的个人健康饮食记录系统，帮助用户管理每日饮食摄入、运动消耗和体重变化。系统采用前后端分离架构，后端使用 Java + Spring Boot + MySQL，前端使用 Angular 框架。

### ✨ 核心功能

- **用户管理**：用户注册、登录、JWT 身份认证
- **饮食记录**：记录每日三餐和加餐，支持增删改查
- **运动记录**：记录每日运动消耗，支持多种运动类型
- **体重记录**：记录体重变化，查看趋势图表
- **统计分析**：按日/周/月维度统计热量摄入与消耗，生成图表
- **话题广场**：用户可以分享饮食情况到公共话题，支持点赞和评论
- **数据隔离**：确保用户间数据独立私有，不会相互干扰

### 🎨 UI 设计

- 马卡龙配色方案
- 清新轻盈的界面风格
- 卡片式设计布局
- 响应式设计，支持移动端

## 🗄️ 数据库配置

### 1. 创建数据库

使用 MySQL 客户端（如 MySQL Workbench、Navicat 或命令行）执行以下 SQL：

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS daily_health 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;

-- 查看数据库
SHOW DATABASES;
```

### 2. 修改数据库连接配置

编辑后端配置文件 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/daily_health?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root           # 改为你的 MySQL 用户名
    password: root           # 改为你的 MySQL 密码
    driver-class-name: com.mysql.cj.jdbc.Driver
```

**注意**：请确保 MySQL 用户具有创建表的权限，系统会自动创建所需的数据表。

## 🔧 常见问题

### 问题 1：后端启动时数据库连接失败

**症状**：控制台显示 `Communications link failure` 或 `Access denied`

**解决方案**：
1. 确认 MySQL 服务是否已启动
2. 检查 `application.yml` 中的数据库配置：
   - 端口是否正确（默认 3306）
   - 用户名和密码是否正确
   - 数据库 `daily_health` 是否已创建
3. 确认 MySQL 用户权限：
```sql
-- 检查用户权限
SHOW GRANTS FOR 'root'@'localhost';
```

### 问题 2：前端 npm install 失败

**症状**：依赖解析错误，显示 `ERESOLVE unable to resolve dependency tree`

**解决方案**：
```bash
# 使用 --legacy-peer-deps 选项
npm install --legacy-peer-deps

# 或者使用 --force
npm install --force
```

**原因**：`ng2-charts` 4.1.1 需要 `@angular/cdk`，而 `@angular/cdk` 的最新版本（21.x）不兼容 Angular 15。在 `package.json` 中我们已指定 `@angular/cdk: "^15.2.0"`，与 Angular 版本匹配。

### 问题 3：前端请求后端跨域

**症状**：浏览器控制台显示 `CORS` 错误

**解决方案**：后端已配置跨域支持，在 `WebSecurityConfig.java` 中：
```java
@Bean
public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration configuration = new CorsConfiguration();
    configuration.setAllowedOrigins(Arrays.asList("*"));
    configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
    configuration.setAllowedHeaders(Arrays.asList("*"));
    // ...
}
```

如果仍有问题，请确认：
- 后端服务是否正常运行在 8080 端口
- 前端服务是否正常运行在 4200 端口
- 前端 `environment` 配置的 API 地址是否正确（当前代码中直接使用 `http://localhost:8080`）

### 问题 4：JWT Token 过期

**症状**：登录后一段时间操作显示 401 未授权

**解决方案**：
- 当前 Token 有效期配置为 24 小时（86400000 毫秒）
- 如需修改，编辑 `application.yml`：
```yaml
jwt:
  expiration: 86400000  # 毫秒，默认24小时
```
- 过期后需要重新登录

### 问题 5：图表不显示

**症状**：体重趋势图或统计图表不显示

**解决方案**：
1. 确认已安装依赖：
```bash
npm list chart.js ng2-charts
```
2. 确认数据量：体重趋势图需要至少 2 条记录才会显示
3. 检查浏览器控制台是否有 JavaScript 错误
